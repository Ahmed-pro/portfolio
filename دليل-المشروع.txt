====================================================
دليل مشروع البورتفوليو - Laravel + Livewire + Filament
====================================================
آخر تحديث: 2026-07-20

--------------------------
1) مكان المشروع والتقنيات
--------------------------
المسار: C:\laragon\www\portfolio

التقنيات المستخدمة:
- Laravel 11
- Filament v5 (لوحة التحكم)
- Livewire 4 (بنمط الكلاسات التقليدي: Class + Blade view)
- Tailwind CSS 3 + Vite
- MySQL (قاعدة بيانات "portfolio" عبر Laragon)
- PHP 8.3.1 (عبر Laragon) - ملاحظة مهمة: جهازك فيه نسخة PHP 7.3 قديمة
  في XAMPP، والمشروع لازم يشتغل بنسخة PHP 8.3 اللي جوه Laragon.

--------------------------------------
2) تشغيل المشروع محلياً (على جهازك)
--------------------------------------
افضل طريقة: افتح برنامج Laragon نفسه:
  - تأكد إن نسخة PHP المختارة في قائمة Laragon هي 8.3.1
  - شغّل Apache/Nginx + MySQL من Laragon
  - المشروع هيبقى متاح تلقائياً على: http://portfolio.test

أو شغّل السيرفر يدوياً من الترمينال (لازم تضيف مسار PHP 8.3 قبل أي أمر):
  في PowerShell أو الـ Terminal اللي بتستخدمه، قبل أي أمر php أو composer
  أو artisan، أضف مسار Laragon's PHP 8.3.1 للـ PATH في نفس الجلسة، مثال
  (Git Bash):
    export PATH="/c/laragon/bin/php/php-8.3.1:$PATH"
    cd /c/laragon/www/portfolio
    php artisan serve --host=127.0.0.1 --port=8000

  ثم افتح: http://127.0.0.1:8000

لو غيرت أي ملف CSS/JS، لازم تعيد البناء:
    npm run build
  (أو "npm run dev" أثناء التطوير عشان يتحدث تلقائياً)

----------------------------------
3) بيانات الدخول للوحة التحكم
----------------------------------
الرابط: /admin  (مثال: http://portfolio.test/admin أو http://127.0.0.1:8000/admin)
البريد: admin@portfolio.test
الباسورد: Password123!

** مهم جداً: غيّر البريد والباسورد دول قبل ما ترفع الموقع على الإنترنت. **
تقدر تعمل يوزر جديد بالأمر التالي (بعد ضبط PATH زي فوق):
    php artisan make:filament-user

------------------------------------------------
4) إيه اللي اتعمل بالظبط - لوحة التحكم (Filament)
------------------------------------------------
- صفحة "About Me" (ManageProfile):
  الاسم، المسمى الوظيفي، الصورة الشخصية، نبذة عني، بيانات التواصل
  (إيميل/تليفون/الموقع)، ملف السيرة الذاتية PDF، روابط السوشيال ميديا
  (جيت هاب/لينكدإن/واتساب...الخ)، وإحصائيات متحركة (زي "3+ سنوات خبرة").

- Skills (المهارات):
  اسم المهارة، التصنيف (Backend/Frontend/Database/DevOps/Design)،
  نسبة الإتقان بشريط تفاعلي (Slider)، وترتيب العرض بالسحب والإفلات.

- Projects (الأعمال):
  عنوان، تصنيف، وصف، صورة، رابط تجربة حية، رابط الكود على جيت هاب،
  تمييز كـ"Featured"، وترتيب بالسحب والإفلات.

- Contact Messages (رسائل التواصل):
  عرض فقط للرسائل الجاية من فورم التواصل في الصفحة الرئيسية،
  مع إمكانية تعليمها "مقروءة" وحذفها. مفيش إنشاء يدوي (مقصود).

----------------------------------------------
5) إيه اللي اتعمل بالظبط - الواجهة العامة (بدون تسجيل دخول)
----------------------------------------------
صفحة واحدة (One Page) على الرابط الرئيسي "/" فيها:
  - Navbar ثابت فيه روابط + زرار Dark/Light Mode (بيتحفظ في المتصفح)
  - Hero: الاسم + المسمى + صورة + أزرار (شوف أعمالي / حمل السيرة الذاتية)
  - إحصائيات متحركة (Counters) تتحرك لما توصلها بالسكرول
  - About: النبذة + معلومات التواصل
  - Skills: بارات مستوى المهارة بتتحرك (Animation) لما تظهر في الشاشة
  - Portfolio: شبكة أعمال قابلة للفلترة حسب التصنيف (عبر Livewire
    بدون إعادة تحميل الصفحة)
  - Contact: فورم تواصل حقيقي بيحفظ الرسائل في قاعدة البيانات، فيه:
      * تحقق من صحة البيانات (Validation)
      * حماية من السبام (Honeypot field + Rate limiting)
  - Footer

ملاحظة: اخترت الميزة المميزة الافتراضية = Dark/Light Mode + أنيميشن
سكرول واحصائيات متحركة (لأن ردك على السؤال ده جه مش واضح). لو عايز
حاجة تانية زي دعم اللغة العربية RTL أو أي إضافة، قولّي.

--------------------------------------------
7) الرفع على استضافة مجانية (Railway / Render)
--------------------------------------------
جهزت الملفات دي عشان الرفع يبقى بسيط:
  - Dockerfile              (يبني الصور + PHP + Apache تلقائياً)
  - docker/entrypoint.sh    (يشغل migrate + storage:link تلقائي عند التشغيل)
  - .dockerignore
  - .env.example            (محدث بإعدادات production)

ملاحظة صراحة: مفيش استضافة "مجانية للأبد بالكامل" مثالية لمشروع
Laravel+MySQL في 2026 زي زمان. أفضل خيارين متاحين:

  * Railway (railway.app) - الأفضل تقنياً لـ Laravel، بيدي رصيد تجريبي
    شهري بسيط بيكفي مشروع بورتفوليو صغير (زيارات قليلة).
  * Render (render.com) - قاعدة بيانات مجانية لأول 90 يوم بس، بعدين
    محتاج ترقية أو تنقل لمكان تاني.

خطوات الرفع على Railway (الموصى بها):
  1. اعمل حساب مجاني على: https://railway.app
  2. ارفع المشروع على GitHub أولاً:
       - اعمل حساب/repo جديد على: https://github.com
       - من مجلد المشروع (بعد ضبط PATH):
           git add -A
           git commit -m "Initial commit"
           git remote add origin <رابط الريبو بتاعك>
           git branch -M main
           git push -u origin main
  3. في Railway: New Project → Deploy from GitHub repo → اختار الريبو
     (هيكتشف الـ Dockerfile تلقائياً ويبني الصورة)
  4. من نفس الـ Project: Add Plugin → MySQL (هيديك بيانات اتصال جاهزة)
  5. من إعدادات السيرفس (Variables) ضيف المتغيرات دي:
       APP_NAME=My Portfolio
       APP_ENV=production
       APP_DEBUG=false
       APP_KEY=  (احصل عليه محلياً بالأمر: php artisan key:generate --show)
       APP_URL=https://<الدومين اللي Railway هيديهولك>
       DB_CONNECTION=mysql
       DB_HOST / DB_PORT / DB_DATABASE / DB_USERNAME / DB_PASSWORD
         (هتلاقيهم جاهزين في تبويب MySQL اللي عملته في نفس الـ Project)
       FILESYSTEM_DISK=public
  6. بعد أول Deploy ناجح، افتح الرابط اللي هيديهولك Railway.
  7. سجل دخول على /admin وابدأ تملأ بياناتك الحقيقية (بدل بيانات
     الديمو اللي موجودة دلوقتي).

نفس الخطوات تقريباً تنفع مع Render، بس بدل "MySQL Plugin" هتستخدم
PostgreSQL المجانية بتاعتهم أو تربط قاعدة بيانات خارجية.
